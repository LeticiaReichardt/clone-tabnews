# clone-tabnews

Projeto ponto a ponto, relembrando tecnologias.

Pontos fortes do projeto:

- Utilizando testes automatizados para então criar as funções necessarias, com a logica que precisa-se chegar no resultado x programado no teste e a criação da função e testada até o resultado x ser alcançado.

- Fremeworks para padronização de alinhamento de código, para ajudar quando o desenvolvimento e feito por uma equipe. Realiza os ajustes no código de forma automatica.

- Utilização de API para acesso ao banco de dados.

- Padrão de testes no estilo trofeu.

- Utilizando banco de dados postgres em container para garantir as configurações exatas necessarias na execução do projeto.

- Configuração de comandos rapidos para execução do projeto e dos testes de forma rapida.

- Criação de deploy conforme ajustes no projeto no Github. Conforme os gits são aprovados o deploy e feito para produção, em caso de erros inesperados é posivel retornar para o deploi ainterios (git anterior) sem afetar o desenvolvmento, as variaveis de banco de dados são diferentes para cada ambinte sendo a troca feita automaticamente no deploy.




TDD, ou Test-Driven Development (Desenvolvimento Orientado a Testes)
